REST API
PART 1: Develop a REST API
Functionalities
Develop a REST API for student management in Python using the Flask library named app.py, with data persistence in a students.csv file. The API should support GET, POST, PUT, PATCH, and DELETE requests. Each student should have fields for id, first name, last name, and age.





HTTP Requests
GET
Retrieve information about a specific student (display all available information):
By their ID; if the provided ID is not found in the CSV file, return an error.
By their last name; if there are multiple students with the same last name, display all of them; if the provided last name is not found in the CSV file, return an error.
Retrieve a list of all students (display all available information).
POST
Create a new student.
The ID field should be automatically generated with an increment of +1.
The POST request body should include first name, last name, and age.
If a non-existing field is passed in the POST body or if no fields are passed at all, return an error.
If any of the fields are missing in the POST body, return an error without writing to the CSV file.
Upon successful request, return information about the added student.
PUT
Update student information by their ID.
It should be possible to update the fields first name, last name, and age.
If the provided ID is not found in the CSV file, return an error.
The PUT request body should include first name, last name, and age accordingly.
If a non-existing field is passed in the PUT body or if no fields are passed at all, return an error.
Upon successful request, return the updated information about the student.
PATCH
Update a student's age by their ID.
It should be possible to update the age field.
If the provided ID is not found in the CSV file, return an error.
The PATCH request body should include the age.
If a non-existing field is passed in the PATCH body or if no fields are passed at all, return an error.
Upon successful request, return the updated information about the student.
DELETE
Delete a student from the CSV file by their ID.
If the provided ID is not found in the CSV file, return an error.
Upon successful request, return a message confirming the successful deletion of the student.

REST API — Управління студентами

ЧАСТИНА 1: Розробка REST API
Функціональність
Розробити REST API для управління студентами на Python з використанням бібліотеки Flask у файлі app.py, із збереженням даних у файлі students.csv. API повинно підтримувати методи GET, POST, PUT, PATCH, DELETE. Кожен студент має поля: id, first name, last name, age.

HTTP-запити
GET
- Отримати інформацію про конкретного студента (показати всю доступну інформацію):
  • за його ID; якщо наданий ID не знайдено у CSV-файлі — повернути помилку;
  • за його прізвищем; якщо є кілька студентів з однаковим прізвищем — показати всіх; якщо прізвища не знайдено у CSV — повернути помилку.
- Отримати список усіх студентів (показати всю доступну інформацію).

POST
- Створити нового студента.
- Поле ID повинно генеруватися автоматично з приростом +1.
- Тіло POST-запиту має містити: first name, last name, age.
- Якщо у POST-потоці передано неіснуюче поле або взагалі не передано жодного поля — повернути помилку.
- Якщо будь-яке з полів відсутнє у тілі POST — повернути помилку без запису у CSV-файл.
- У разі успіху повернути інформацію про доданого студента.

PUT
- Оновити інформацію про студента за його ID.
- Має бути можливість оновити поля: first name, last name, age.
- Якщо наданий ID не знайдено у CSV — повернути помилку.
- Тіло PUT-запиту повинно містити відповідно first name, last name, age (усі три поля).
- Якщо у PUT тілі передано неіснуюче поле або не передано жодного поля — повернути помилку.
- У разі успіху повернути оновлену інформацію про студента.

PATCH
- Оновити вік (age) студента за його ID.
- Має бути можливість оновити поле age.
- Якщо наданий ID не знайдено у CSV — повернути помилку.
- Тіло PATCH-запиту повинно містити поле age.
- Якщо у PATCH тілі передано неіснуюче поле або не передано жодного поля — повернути помилку.
- У разі успіху повернути оновлену інформацію про студента.

DELETE
- Видалити студента з CSV-файлу за його ID.
- Якщо наданий ID не знайдено у CSV — повернути помилку.
- У разі успіху повернути повідомлення, що підтверджує успішне видалення студента.

ЧАСТИНА 2: Створення test_requests.py
Функціональність
Створити файл test_requests.py для перевірки створеного REST API. У цьому файлі продемонструвати роботу всіх методів за допомогою бібліотеки requests у такій послідовності:
1) Отримати всіх існуючих студентів (GET).
2) Створити трьох студентів (POST).
3) Отримати інформацію про всіх існуючих студентів (GET).
4) Оновити вік другого студента (PATCH).
5) Отримати інформацію про другого студента (GET).
6) Оновити ім’я, прізвище та вік третього студента (PUT).
7) Отримати інформацію про третього студента (GET).
8) Отримати всіх існуючих студентів (GET).
9) Видалити першого студента (DELETE).
10) Отримати всіх існуючих студентів (GET).
11) Відобразити результати виконання в консолі та записати їх у файл results.txt.
